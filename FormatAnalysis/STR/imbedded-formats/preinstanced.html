Preinstanced Asset Format (EARS_MESH) - Complete Implementation Guide

Version: 2.0

Last Updated: 2025-02-17

Author(s): samarixum (based on reverse-engineering research)

Overview

  • Format Name: Simpsons Game Preinstanced Asset / EARS_MESH
  • Common File Extension(s): .dff.preinstanced, .rws.preinstanced (embedded in STR sections)
  • Purpose/Domain: Defines 3D model geometry, multiple UV maps, and texture name associations for assets in The Simpsons Game (PS3).
  • Originating Application/System: A heavily modified version of RenderWare customized by EA for The Simpsons Game.
  • Format Type: Binary, sequential chunk-based (no global header).
  • General Structure: File is composed of sequential chunks: texture headers + texture names, TLFD delimiters, mesh chunks with geometry, and an EOF marker. No single magic number identifies the format; instead, look for characteristic signatures.
  • Endianness: Mixed-Endian. Almost all data is Big-Endian, except for FaceDataOff and MeshDataSize in mesh chunk headers (Little-Endian).

Parsing Algorithm Overview

  1. Scan file sequentially from byte 0 to EOF
  2. Maintain a "pending_texture_list" (empty initially)
  3. Maintain a "texture_snapshots" array (empty initially)
  4. Maintain a "meshes" array (empty initially)
  5. For each detected signature:
    • Texture Header → Extract texture name → Push to pending_texture_list
    • TLFD Marker → Save pending_texture_list to texture_snapshots → Clear pending_texture_list
    • Mesh Chunk → Parse mesh data → Link with most recent texture_snapshots → Push to meshes
    • EOF Marker → Stop parsing
  6. Return meshes array with texture bindings

Identification Signatures

Texture Header (3 Known Variants)

These 16-byte signatures precede a texture name string.

  • 02 11 01 00 02 00 00 00 14 00 00 00 2D 00 02 1C (Type A)
  • 02 11 01 00 02 00 00 00 18 00 00 00 2D 00 02 1C (Type B)
  • 02 11 01 00 02 00 00 00 10 00 00 00 2D 00 02 1C (Type C)

Interpretation: All three variants have the same format. The differences at offsets 8-11 (0x14, 0x18, 0x10) may indicate offset or size variants, but all point to a texture name immediately following.

Texture List Delimiter (TLFD)

54 4C 46 44 (ASCII "TLFD")

Function: Signals end of a block of texture names. The next mesh chunk found will use the accumulated texture list up to this point.

Mesh Chunk Signature

33 EA 00 00 followed by 4 bytes (wildcard), then 2D 00 02 1C

Pattern: 33 EA 00 00 ?? ?? ?? ?? 2D 00 02 1C

Function: Marks the start of a mesh chunk header.

EOF Marker

16 EA 00 00 05 00 00 00 2D 00 02 1C 01 00 00 00 00

Function: Signals end of valid file data. Parsing should stop here.

Texture Name Extraction

Location: Immediately after a Texture Header signature (16 bytes).

Format: Null-terminated ASCII string.

Algorithm:

  1. Position reader after the 16-byte Texture Header
  2. Read bytes until null terminator (0x00) is found
  3. Convert bytes to ASCII string
  4. Sanitize: Remove non-alphanumeric characters except ._-
  5. If result is empty, use fallback: texture_at_0x<HEX_OFFSET>
  6. Add string to pending_texture_list

Mesh Chunk Header Structure

Location: Starts at the byte after the mesh chunk signature (33 EA 00 00 ?? ?? ?? ?? 2D 00 02 1C).

The mesh chunk header uses a mixed-endian approach. While most architectural and count fields are Big-Endian (PS3), some core size and offset fields are Little-Endian.

Header Layout

Offset (Abs) Offset (Rel) Size Type Endian Field Name
0x00 - 4 UInt32 BE Signature (33 EA 00 00)
0x04 - 4 UInt32 BE Unknown
0x08 - 4 UInt32 BE Signature Verify (2D 00 02 1C)
0x0C 0x00 4 UInt32 BE Padding / mVersion
0x10 0x04 4 UInt32 LE FaceDataOff
0x14 0x08 4 UInt32 LE MeshDataSize
0x18 0x0C - - - MeshChunkStart (Header Base)

Navigation Table and Sub-Meshes

Following MeshChunkStart (0x18 abs), there is a 28-byte block of padding before the table counts begin.

Offset (Rel to Base) Size Type Endian Field Name
0x1C 4 UInt32 BE TableCount
0x20 4 UInt32 BE SubmeshCount

Sub-Mesh Table Entries

Located after the Table entries (each 8 bytes, total TableCount * 8). Each Sub-Mesh entry is 12 bytes.

Offset (Sub-Mesh) Size Type Field Name Description
0x00 8 bytes[8] Unknown Skipped.
0x08 4 UInt32 SubmeshRelOffset Offset to Sub-Mesh header (relative to MeshChunkStart).

Vertex Information Structure

Located at MeshChunkStart + SubmeshRelOffset + 12 is a pointer VertInfoRelOffset. Following that pointer (relative to MeshChunkStart):

Offset (VertInfo) Type Endian Field Name Description
0x00 UInt32 BE VertTotalSize Total size of vertex buffer.
0x04 UInt32 BE VertStride Bytes per vertex (stride).
0x0C UInt32 BE VertStartRel Offset to vertex data (relative to FaceDataOff).
0x24 UInt32 BE FaceByteLen Total size of face index buffer (bytes).
0x2C UInt32 BE FaceStartRel Offset to face data (relative to FaceDataOff).

Header Traversal Algorithm (Unified)

Pseudocode for finding the mesh header end:


function find_header_end(mesh_chunk_start):
    // We know the header contains offset/size fields at variable positions
    // Scan forward from chunk start until we find non-sequential data
    
    version = read_u32_be(mesh_chunk_start + 0x00)
    data_table_count = read_u32_be(mesh_chunk_start + 0x04)
    
    // Skip data table
    header_pos = mesh_chunk_start + 0x08 + (data_table_count * 8)
    
    // Read vertex info
    vert_info_count = read_u32_be(header_pos)
    vert_chunk_size = read_u32_be(header_pos + 0x04)
    vert_chunk_total = read_u32_be(header_pos + 0x08)
    vert_start = read_u32_be(header_pos + 0x0C)
    face_start = read_u32_be(header_pos + 0x10)
    face_data_byte_len = read_u32_be(header_pos + 0x14)
    
    // The actual mesh data starts ~32-48 bytes after vertex info begins
    // Conservative approach: skip to known offset
    header_end = header_pos + 0x24  // Typical header size after vertex info
    
    return header_end

Mesh Data Extraction

Vertex Buffer

Location: mesh_chunk_start + FaceDataOff + VertexStart (note: FaceDataOff is Little-Endian)

Vertex Count: VertChunkTotalSize / VertChunkSize

Vertex Structure (per vertex, Big-Endian):

Offset Size Type Field Description
0x00 12 float[3] Position (X, Y, Z) 3D vertex coordinates.
0x0C 8 bytes[8] Unknown Likely normals or other attributes.
0x14 4 float UV0 U (Primary Texture) U coordinate for main texture map.
0x18 4 float UV0 V (Primary Texture) V coordinate. Must flip: (1.0 - V) for proper rendering.
0x1C 4 float UV1 U (Secondary/CM Texture) U coordinate for secondary (color/environment map).
0x20 4 float UV1 V (Secondary/CM Texture) V coordinate. Must flip: (1.0 - V).
0x24+ VertChunkSize - 36 bytes[N] Additional Attributes Vertex colors, bone weights, normals, or tangents. Layout varies.

Face/Index Buffer

Location: mesh_chunk_start + FaceDataOff + FaceStart

Index Count: FaceDataByteLength / 2

Index Structure (per index):

  • 2 bytes UInt16 (Big-Endian): Vertex index
  • Special Value: 0xFFFF (65535) is a "strip cut" delimiter. This marks the end of one triangle strip and the start of another. It should not be rendered as an index.

Triangle Strip Conversion Algorithm:


function build_triangle_list(indices):
    triangles = []
    current_strip = []
    
    for each index in indices:
        if index == 0xFFFF:  // Strip cut
            if current_strip.length >= 3:
                // Convert triangle strip to individual triangles
                for i = 0 to current_strip.length - 3:
                    if i % 2 == 0:
                        triangles.append({
                            v0: current_strip[i],
                            v1: current_strip[i+1],
                            v2: current_strip[i+2]
                        })
                    else:
                        // Reverse winding for even-indexed triangles
                        triangles.append({
                            v0: current_strip[i],
                            v1: current_strip[i+2],
                            v2: current_strip[i+1]
                        })
            current_strip = []
        else:
            current_strip.append(index)
    
    return triangles

Complete Parsing Example (Pseudocode)


function parse_preinstanced(file_data):
    meshes = []
    pending_textures = []
    current_texture_snapshot = null
    
    pos = 0
    while pos < len(file_data):
        // Check for Texture Header (3 variants)
        if match_pattern(file_data, pos, "02 11 01 00 02 00 00 00 14 00 00 00 2D 00 02 1C") or
           match_pattern(file_data, pos, "02 11 01 00 02 00 00 00 18 00 00 00 2D 00 02 1C") or
           match_pattern(file_data, pos, "02 11 01 00 02 00 00 00 10 00 00 00 2D 00 02 1C"):
            
            pos += 16  // Skip signature
            
            // Extract texture name
            name_start = pos
            while file_data[pos] != 0x00 and pos < len(file_data):
                pos += 1
            
            name = sanitize(file_data[name_start:pos])
            pending_textures.append(name)
            pos += 1  // Skip null terminator
        
        // Check for TLFD marker
        else if match_pattern(file_data, pos, "54 4C 46 44"):
            current_texture_snapshot = copy(pending_textures)
            pending_textures = []
            pos += 4
        
        // Check for EOF marker
        else if match_pattern(file_data, pos, "16 EA 00 00 05 00 00 00 2D 00 02 1C 01 00 00 00 00"):
            break  // End of valid file
        
        // Check for Mesh Chunk
        else if match_pattern(file_data, pos, "33 EA 00 00 ?? ?? ?? ?? 2D 00 02 1C"):
            mesh_start = pos
            pos += 16  // Skip signature
            
            // Parse header
            version = read_u32_be(pos)
            data_table_count = read_u32_be(pos + 0x04)
            
            header_offset = pos + 0x08 + (data_table_count * 8)
            vert_info_count = read_u32_be(header_offset)
            vert_chunk_size = read_u32_be(header_offset + 0x04)
            vert_chunk_total = read_u32_be(header_offset + 0x08)
            vert_start = read_u32_be(header_offset + 0x0C)
            face_start = read_u32_be(header_offset + 0x10)
            face_data_byte_len = read_u32_be(header_offset + 0x14)
            face_data_off_le = read_u32_le(header_offset + 0x18)
            mesh_data_size_le = read_u32_le(header_offset + 0x1C)
            
            // Extract vertex data
            vertex_count = vert_chunk_total / vert_chunk_size
            vert_data_offset = mesh_start + face_data_off_le + vert_start
            vertices = []
            
            for v = 0 to vertex_count - 1:
                v_offset = vert_data_offset + (v * vert_chunk_size)
                
                position = read_float3_be(v_offset + 0x00)
                tex0_u = read_float_be(v_offset + 0x14)
                tex0_v = 1.0 - read_float_be(v_offset + 0x18)  // Flip V
                tex1_u = read_float_be(v_offset + 0x1C)
                tex1_v = 1.0 - read_float_be(v_offset + 0x20)  // Flip V
                
                vertices.append({
                    position: position,
                    uv0: {u: tex0_u, v: tex0_v},
                    uv1: {u: tex1_u, v: tex1_v}
                })
            
            // Extract face/index data
            index_count = face_data_byte_len / 2
            face_data_offset = mesh_start + face_data_off_le + face_start
            indices = []
            
            for i = 0 to index_count - 1:
                index = read_u16_be(face_data_offset + (i * 2))
                if index != 0xFFFF:
                    indices.append(index)
            
            // Convert triangle strips
            triangles = build_triangle_list(indices)
            
            mesh = {
                vertices: vertices,
                triangles: triangles,
                textures: current_texture_snapshot
            }
            
            meshes.append(mesh)
            
            // Skip to next chunk (rough estimate)
            pos = mesh_start + mesh_data_size_le + 32
        
        else:
            pos += 1  // Advance byte-by-byte until next signature
    
    return meshes

Validation Rules

  • Vertex Count: Must be > 0. If 0, mesh is invalid.
  • Triangle Count: Must be > 0. If all indices are 0xFFFF, mesh is invalid.
  • Index Bounds: All non-delimiter indices must be < vertex_count.
  • VertChunkSize: Should match actual stride in vertex data (typically 36, 48, or 64).
  • Endianness Check: If vert_chunk_size appears impossibly large (> 1000), may have endianness error. Verify byte order.
  • Texture Count: Should match or exceed the number of vertices/triangles that reference textures.

Known Limitations & Unknowns

  • mDataTable: Purpose unclear. Most parsers skip it entirely.
  • FaceDataOff / MeshDataSize: Exact semantics unclear. Used as offsets but specific meaning not documented in reverse-engineering.
  • Full Vertex Stride: Only position and two UV pairs are documented. Additional fields (normals, vertex colors, bone info) exist but layout varies.
  • Mipmap Handling: Not addressed in this format. Assume single resolution.
  • Bone/Skinning Data: May be embedded in the "unknown" vertex stride bytes. Not extracted by current parsers.
