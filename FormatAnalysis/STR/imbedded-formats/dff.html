Preinstanced Mesh (.dff)

Files with extension .dff.preinstanced in this game engine use a custom stream format containing texture associations and mixed-endian mesh chunks.

Stream Structure

The file is scanned as a linear stream of events. Key markers include:

  • Texture Headers: 02 11 01 00 02 00 00 00 ... 2D 00 02 1C. Contains texture names.
  • TLFD Marker: The ASCII string TLFD. Used to associate previous textures with the next mesh.
  • Mesh Signature: 33 EA 00 00 followed by 2D 00 02 1C at offset +8.

Mesh Chunk

The mesh chunk contains geometry data. Note: While the file is generally Big Endian (PS3), specific fields in the mesh header are Little Endian.

Offset Type Endian Name Description
0x00 Byte[4] BE Signature 33 EA 00 00
0x04 Byte[4] BE Unknown
0x08 Byte[4] BE Verify 2D 00 02 1C
0x0C Byte[4] BE Padding
0x10 UInt32 LE FaceDataOffset Offset to face indices (Little Endian).
0x14 UInt32 LE MeshDataSize Size of the mesh data (Little Endian).
0x18 - - MeshChunkStart Base offset for relative pointers below.

Following MeshChunkStart (and after 28 bytes of padding):

Offset (Rel) Type Name Description
0x1C UInt32 TableCount Number of table entries (skipped).
0x20 UInt32 SubmeshCount Number of sub-meshes.

Sub-Mesh Entry

Located after the Table entries (8 bytes each). Each Sub-Mesh entry is 12 bytes.

Offset Type Name Description
0x00 Byte[8] Unknown Skipped.
0x08 UInt32 SubmeshRelOffset Offset to Sub-Mesh header (relative to MeshChunkStart).

Vertex Information

Located at MeshChunkStart + SubmeshRelOffset + 12 is a pointer VertInfoRelOffset. Following that pointer:

Offset Type Name Description
0x00 UInt32 VertTotalSize Total size of vertex buffer.
0x04 UInt32 VertStride Bytes per vertex.
0x0C UInt32 VertStartRel Offset to vertex data (relative to FaceDataOffset base).
0x24 UInt32 FaceByteLen Total size of face index buffer (bytes).
0x2C UInt32 FaceStartRel Offset to face data (relative to FaceDataOffset base).
